# Knowledge Base

This directory is the operational knowledge base for the repository.

Use it as the detailed complement to the top-level README.

## Contents

- `architecture.md`: current contract and runtime architecture.
- `ci.md`: what GitHub Actions validates and why.
- `local-development.md`: how to compile, run a local chain, and deploy locally.
- `testing.md`: unit and external-network integration testing workflow.
- `ai-workflows.md`: how to use the `.github` customizations and what they are for.

## Maintenance rule

When a workflow changes in code, update the matching document here in the same change.
